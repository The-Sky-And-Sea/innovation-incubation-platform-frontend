# 账号与认证系统 - 规格说明

## 1. 模块概述

账号与认证系统是创新创业孵化载体管理平台的根基，提供三端（企业端、载体端、政务端）用户的注册、登录和权限管理功能。

**核心目标**：
- 支持企业、载体、政务三种角色的注册与登录
- 基于 JWT 的身份认证机制
- 基于 Casbin 的 RBAC 权限控制

## 2. 技术栈

### 前端
- React 19 + TypeScript
- React Router DOM v6
- Zustand 状态管理
- Ant Design 6 组件库

### 后端
- Go + Gin 框架
- GORM + PostgreSQL
- Casbin RBAC
- JWT 认证

## 3. 功能详情

### 3.1 注册功能

**支持的角色**：
| 角色 | 标识 | 适用场景 |
|------|------|----------|
| 企业 | enterprise | 入驻申请、政策申报、材料上传 |
| 载体 | carrier | 入驻审核、绩效填报 |
| 政务 | government | 政策发布、终审 |

**注册流程**：
1. 选择注册身份（企业/载体/政务）
2. 填写账号信息（手机号、邮箱、密码）
3. 填写主体资料
   - 企业：企业名称、统一社会信用代码、行业、规模、地址
   - 载体：载体名称、载体类型、所在区域
   - 政务：政务人员姓名、所属部门
4. 确认提交

### 3.2 登录功能

**登录方式**：
- 手机号 + 密码
- 企业额外支持统一社会信用代码 + 密码

**登录后跳转**：
- 企业 → `/enterprise/dashboard`
- 载体 → `/carrier/dashboard`
- 政务 → `/gov/dashboard`

### 3.3 权限控制

**守卫模式**：
- `GuestGuard`：未登录用户访问（登录页、注册页）
- `AuthGuard`：需要登录才能访问
- `RoleGuard`：需要特定角色才能访问

**路由结构**：
```
/login, /register        → GuestGuard
/enterprise/*            → RoleGuard: enterprise
/carrier/*              → RoleGuard: carrier
/gov/*                 → RoleGuard: government
```

## 7. 安全考虑

1. **密码存储**：使用 bcrypt 哈希存储，不明文保存
2. **Token 机制**：JWT Access Token + Refresh Token
3. **权限控制**：Casbin 基于角色的访问控制
4. **输入校验**：手机号格式、邮箱格式、密码强度
