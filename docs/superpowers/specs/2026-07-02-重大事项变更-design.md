# 重大事项变更模块 - 规格说明

## 1. 模块概述

重大事项变更模块用于管理企业在入驻后的信息变更，如企业名称、信用代码、行业、规模、地址、法定代表人等关键信息的变更申请和审核流程。

**核心目标**：
- 企业端：提交变更申请、查看变更记录
- 载体端：审核变更申请
- 政务端：终审变更申请

## 2. 业务流程

### 2.1 变更申请流程

```
企业提交变更申请
    ↓
载体审核（初审）
    ↓
政务审核（终审）
    ↓
变更成功 / 被拒绝
```

### 2.2 可变更项目

| 变更类型 | 说明 |
|----------|------|
| 企业名称 | 企业名称变更 |
| 统一社会信用代码 | 信用代码变更 |
| 所属行业 | 行业类型变更 |
| 企业规模 | 规模变更（大型/中型/小型/微型企业） |
| 企业地址 | 注册地址变更 |
| 法定代表人 | 法定代表人变更 |
| 入孵协议文件 | 入驻协议文件更新 |

### 2.3 变更状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 草稿 | draft | 企业保存但未提交 |
| 待审核 | pending | 已提交，等待载体审核 |
| 载体审核中 | carrier_review | 载体正在审核 |
| 政务审核中 | gov_review | 政务正在审核 |
| 已通过 | approved | 变更已生效 |
| 已拒绝 | rejected | 变更被拒绝 |
| 已退回 | returned | 被退回修改 |

## 3. 功能详情

### 3.1 前置条件

**重要**：企业必须先完成入驻申请并通过载体审核后，才能进行重大事项变更。

入驻状态判断条件：
- `status` = "approved"（已通过审核）
- `incubate_status` = "in_incubation"（在孵中）

### 3.2 企业端 - 变更管理

**功能**：企业提交和管理重大事项变更申请

**页面**：`/enterprise/changes`

**功能点**：
- 查看变更记录列表
- 新建变更申请
- 查看变更详情
- 被退回时重新编辑提交

**操作限制**：
- 未入驻企业无法新建变更申请
- 未入驻时显示提示信息："请先申请入驻并通过载体审核后，才能进行重大事项变更"
- 按钮禁用状态

### 3.3 载体端 - 变更审核

**功能**：载体审核企业提交的变更申请

**页面**：`/carrier/changes`

**功能点**：
- 查看待审核变更列表
- 查看变更详情（原值 vs 新值）
- 审核操作：通过 / 拒绝 / 退回

### 3.4 政务端 - 变更审批

**功能**：政务终审重大事项变更申请

**页面**：（集成在政务审批页面中）

**功能点**：
- 查看待审批变更列表
- 查看变更详情和载体审核意见
- 终审操作：通过 / 拒绝

## 7. 业务规则

### 7.1 入驻状态校验
- 前端：页面加载时调用获取入驻记录 API
- 后端：提交变更申请前校验企业是否有有效入驻记录
- 校验条件：`status = 'approved' AND incubate_status = 'in_incubation'`

### 7.2 变更类型处理
- 普通字段：直接显示原值和新值
- 文件类型：显示文件名和文件ID

### 7.3 审核流程
- 载体审核后可选择：批准、拒绝、退回
- 政务审核后可选择：批准、拒绝
- 退回后企业可重新编辑提交
