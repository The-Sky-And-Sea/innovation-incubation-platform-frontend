# 绩效考核模块 - 规格说明

## 1. 模块概述

绩效考核模块用于管理和评估载体的运营绩效，实现从模板配置、考核启动、载体填报到政务评分的完整流程。

**核心目标**：
- 政务端：配置考核模板、启动考核、评分
- 载体端：填写绩效数据、提交考核

## 2. 业务流程

### 2.1 绩效考核流程

```
政务配置考核模板
    ↓
政务启动考核
    ↓
载体填写绩效数据
    ↓
载体提交考核
    ↓
政务评分
    ↓
考核完成
```

### 2.2 考核状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 待填报 | pending | 考核已启动，载体未填报 |
| 填报中 | filling | 载体正在填写 |
| 待评分 | submitted | 载体已提交，等待政务评分 |
| 已评分 | scored | 政务已评分 |
| 已过期 | expired | 超过截止日期 |

### 2.3 考核模板内容

考核模板包含多个考核指标：
- 入驻企业数量
- 毕业企业数量
- 税收贡献
- 带动就业人数
- 知识产权数量
- 科技型企业数量
- 融资情况
- 其他自定义指标

每个指标包含：
- 指标名称
- 指标说明
- 满分值
- 评分标准

## 3. 功能详情

### 3.1 政务端 - 绩效模板配置

**功能**：政务配置绩效考核模板

**页面**：`/gov/performances`

**功能点**：
- 查看已有模板
- 创建新模板
- 编辑模板内容（添加/删除/修改指标）
- 删除模板

**模板内容**：
- 模板名称
- 考核年份
- 考核指标列表（名称、说明、满分、评分标准）

### 3.2 政务端 - 启动考核

**功能**：政务启动新一轮绩效考核

**功能点**：
- 选择考核模板
- 选择参与考核的载体
- 设置填报截止日期
- 发送通知给载体

### 3.3 政务端 - 绩效评分

**功能**：政务对载体提交的绩效数据进行评分

**页面**：`/gov/performances`

**功能点**：
- 查看待评分考核列表
- 查看载体填报的绩效数据
- 对每个指标进行评分
- 填写评语
- 提交评分

### 3.4 载体端 - 绩效填报

**功能**：载体填写绩效考核数据

**页面**：`/carrier/performances`

**功能点**：
- 查看待填报考核列表
- 填写各项绩效指标数据
- 上传证明材料
- 提交考核
- 查看历史填报记录

## 7. 业务规则

### 7.1 模板配置
- 每个模板包含多个指标
- 每个指标有名称、说明、满分值

### 7.2 考核启动
- 必须选择已存在的模板
- 必须选择参与考核的载体
- 必须设置填报截止日期

### 7.3 评分规则
- 每个指标单独评分
- 总分由各指标得分汇总
- 评分后不可修改
