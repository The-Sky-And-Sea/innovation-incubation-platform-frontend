# 政策管理模块 - 实现计划

## 任务概述

本计划覆盖政策管理模块的完整实现，包括政务端政策发布、载体端申报初审、企业端申报的政策全流程管理。

## 实现顺序

1. **后端数据模型 (T1.1-T1.5)** → 定义数据结构
2. **后端政策 API (T2.1-T2.6)** → 政策管理接口
3. **后端申报 API (T3.1-T3.6)** → 申报审核接口
4. **政务端政策管理 (T5.1-T5.6)** → 政策发布功能
5. **政务端申报审批 (T6.1-T6.5)** → 终审功能
6. **载体端政策列表 (T7.1-T7.4)** → 查看政策
7. **载体端申报审核 (T8.1-T8.5)** → 初审功能
8. **企业端政策列表 (T9.1-T9.7)** → 浏览和申报
9. **企业端申报功能 (T10.1-T10.4)** → 提交申报

## 验收标准

- [ ] 政务可以创建、编辑、发布、下架政策
- [ ] 企业和载体可以查看已发布政策
- [ ] 超过截止日期的政策企业不可申报
- [ ] 企业可以提交政策申报
- [ ] 企业可以查看自己的申报记录和状态
- [ ] 载体可以初审政策申报
- [ ] 载体可以退回申报让企业修改
- [ ] 政务可以终审政策申报
- [ ] 申报状态正确流转
- [ ] 政策状态正确控制可见性

## 依赖关系

- 政策管理依赖 Policy 模型和后端 API
- 申报审批依赖 PolicyApplication 模型和后端 API
- 企业政策列表依赖企业政策 API 和申报 API
