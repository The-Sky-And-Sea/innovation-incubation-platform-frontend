# 绩效考核模块 - 实现计划

## 任务概述

本计划覆盖绩效考核模块的完整实现，包括政务端模板配置、考核启动、评分功能和载体端绩效填报功能。

- [ ] **T1.1** 定义 `PerformanceTemplate` 模型（绩效模板表）
- [ ] **T1.2** 定义 `PerformanceCampaign` 模型（考核活动表）
- [ ] **T1.3** 定义 `PerformanceData` 模型（绩效数据表）
- [ ] **T1.4** 定义考核状态枚举
- [ ] **T1.5** 定义指标数据结构

## 实现顺序

1. **后端数据模型 (T1.1-T1.5)** → 定义数据结构
2. **后端模板 API (T2.1-T2.4)** → 模板管理接口
3. **后端考核 API (T3.1-T3.7)** → 考核和评分接口
4. **政务端绩效管理 (T4.1-T4.7)** → 模板、启动、评分功能
5. **载体端绩效填报 (T5.1-T5.8)** → 填报功能
6. **评分优化 (T6.1-T6.3)** → 完善评分功能
7. **通知功能 (T7.1-T7.2)** → 考核启动和评分通知

## 验收标准

- [ ] 政务可以创建、编辑、删除考核模板
- [ ] 政务可以启动考核并指定载体
- [ ] 政务可以查看待评分列表
- [ ] 政务可以查看载体填报的绩效数据
- [ ] 政务可以给每个指标评分并提交
- [ ] 载体可以查看待填报考核列表
- [ ] 载体可以填写绩效数据
- [ ] 载体可以上传证明材料
- [ ] 载体可以提交考核
- [ ] 载体可以查看评分结果
- [ ] 考核状态正确流转
- [ ] 总分正确计算

## 依赖关系

- 模板管理依赖 PerformanceTemplate 模型
- 考核启动依赖模板和载体选择
- 评分功能依赖绩效数据和评分 API
- 载体填报依赖考核活动和模板指标
