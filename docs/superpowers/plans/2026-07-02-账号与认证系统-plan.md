# 账号与认证系统 - 实现计划

## 任务概述

本计划覆盖账号与认证系统的完整实现，包括前端登录/注册页面、后端认证 API、JWT 认证中间件和 RBAC 权限控制。

## 任务清单

### Phase 1: 前端 - 登录页面

- [ ] **T1.1** 创建 `pages/auth/LoginPage.tsx` 登录页面组件
- [ ] **T1.2** 实现手机号/统一社会信用代码 + 密码登录表单
- [ ] **T1.3** 添加登录成功后根据角色跳转到对应仪表盘
- [ ] **T1.4** 集成 `authStore` 状态管理

### Phase 2: 前端 - 注册页面

- [ ] **T2.1** 创建 `pages/auth/RegisterPage.tsx` 注册页面组件
- [ ] **T2.2** 实现分步注册向导（选择身份 → 账号信息 → 主体资料 → 确认提交）
- [ ] **T2.3** 实现企业/载体/政务三种角色的不同表单字段
- [ ] **T2.4** 集成注册 API 并处理成功/失败反馈

### Phase 3: 前端 - 路由守卫

- [ ] **T3.1** 实现 `GuestGuard` 组件 - 未登录用户访问控制
- [ ] **T3.2** 实现 `AuthGuard` 组件 - 登录用户访问控制
- [ ] **T3.3** 实现 `RoleGuard` 组件 - 角色权限控制
- [ ] **T3.4** 配置 `router/index.tsx` 路由守卫

### Phase 4: 前端 - 状态管理

- [ ] **T4.1** 完善 `store/authStore.ts` 状态管理
- [ ] **T4.2** 实现 token 存储和自动刷新
- [ ] **T4.3** 实现用户信息获取和角色判断

### Phase 5: 后端 - 认证 API

- [ ] **T5.1** 实现 `POST /api/v1/auth/register` 注册接口
- [ ] **T5.2** 实现 `POST /api/v1/auth/login` 登录接口
- [ ] **T5.3** 实现 `POST /api/v1/auth/logout` 登出接口
- [ ] **T5.4** 实现 `GET /api/v1/auth/me` 获取当前用户信息

### Phase 6: 后端 - JWT 中间件

- [ ] **T6.1** 实现 JWT Token 生成和验证
- [ ] **T6.2** 实现 Token 刷新机制
- [ ] **T6.3** 实现 `middleware/auth.go` 认证中间件

### Phase 7: 后端 - RBAC 权限

- [ ] **T7.1** 集成 Casbin RBAC 框架
- [ ] **T7.2** 实现 `middleware/rbac.go` 权限中间件
- [ ] **T7.3** 配置基于角色的路由权限

## 实现顺序

1. **后端 API (T5.1-T5.4)** → 提供接口供前端调用
2. **前端登录页面 (T1.1-T1.4)** → 优先实现登录功能
3. **JWT 中间件 (T6.1-T6.3)** → 后端认证基础设施
4. **路由守卫 (T3.1-T3.4)** → 前端权限控制
5. **状态管理 (T4.1-T4.3)** → 登录状态持久化
6. **注册页面 (T2.1-T2.4)** → 注册功能
7. **RBAC 权限 (T7.1-T7.3)** → 后端权限控制

## 验收标准

- [ ] 前端登录页面可正常登录，登录后跳转到对应仪表盘
- [ ] 前端注册页面可完成三端用户的注册
- [ ] 未登录用户无法访问受保护路由
- [ ] 跨角色用户无法访问其他角色的页面
- [ ] 后端 API 正确返回用户信息
- [ ] JWT Token 正确生成和验证
- [ ] RBAC 权限正确拦截未授权请求

## 依赖关系

- 登录页面依赖后端登录 API
- 路由守卫依赖 authStore 状态管理
- 注册页面依赖后端注册 API
