# 入驻管理模块 - 实现计划

## 任务概述

本计划覆盖入驻管理模块的完整实现，包括企业端入驻申请、载体端审核、政务端审批的完整业务流程。

## 实现顺序

1. **后端数据模型 (T8.1-T8.4)** → 定义数据结构
2. **后端 API (T7.1-T7.5)** → 提供接口
3. **载体浏览 (T1.1-T1.4)** → 企业端入口
4. **入驻申请 (T2.1-T2.6)** → 企业端核心功能
5. **载体审核 (T3.1-T3.5)** → 载体端核心功能
6. **企业管理 (T4.1-T4.4)** → 载体端查看功能
7. **政务审批 (T5.1-T5.4)** → 政务端核心功能
8. **毕业/退出 (T6.1-T6.4)** → 政务端管理功能

## 验收标准

- [ ] 企业可以浏览载体列表并申请入驻
- [ ] 企业可以提交入驻申请材料
- [ ] 企业可以查看自己的入驻记录和状态
- [ ] 载体可以查看待审核申请列表
- [ ] 载体可以审核通过/拒绝/退回入驻申请
- [ ] 载体可以查看已入驻企业列表
- [ ] 政务可以查看所有入驻申请
- [ ] 政务可以终审通过/拒绝入驻申请
- [ ] 政务可以办理企业毕业/退出备案
- [ ] 入驻状态正确流转（草稿→待审核→载体审核→政务审核→通过/拒绝）
- [ ] 审核记录完整保存

## 依赖关系

- 入驻申请页面依赖后端入驻 API 和文件上传组件
- 审核页面依赖后端审核 API
- 审批页面依赖后端审批 API
