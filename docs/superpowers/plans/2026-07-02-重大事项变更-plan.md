# 重大事项变更模块 - 实现计划

## 任务概述

本计划覆盖重大事项变更模块的完整实现，包括企业端变更申请、载体端审核、政务端审批的全流程。

## 任务清单

- [ ] **T1.1** 定义 `MajorChange` 模型
- [ ] **T1.2** 定义变更状态枚举
- [ ] **T1.3** 定义变更类型常量

### Phase 2: 后端 API

- [ ] **T2.1** 实现获取变更类型列表接口

- [ ] **T2.3** 实现获取变更记录列表接口
- [ ] **T2.4** 实现获取变更详情接口
- [ ] **T2.5** 实现更新变更申请接口
- [ ] **T2.6** 实现载体审核变更接口

## 实现顺序

1. **后端数据模型 (T1.1-T1.3)** → 定义数据结构
2. **后端 API (T2.1-T2.8)** → 提供接口，特别是入驻状态校验
3. **前端 API (T3.1-T3.2)** → 调用后端接口
4. **企业端变更管理 (T4.1-T4.7)** → 核心功能
5. **载体端变更审核 (T5.1-T5.5)** → 审核功能
6. **政务端变更审批 (T6.1-T6.3)** → 终审功能
7. **样式优化 (T7.1-T7.3)** → 优化用户体验

## 验收标准

- [ ] 未入驻企业无法提交变更申请（前端禁用 + 后端校验）
- [ ] 未入驻企业看到提示信息
- [ ] 企业可以查看变更记录列表
- [ ] 企业可以提交变更申请
- [ ] 载体可以审核变更申请
- [ ] 政务可以终审变更申请
- [ ] 变更状态正确流转
- [ ] 变更记录正确保存

## 依赖关系

- 变更管理页面依赖 getMyIncubation API 校验入驻状态
- 变更管理页面依赖 changes API 进行 CRUD 操作
- 变更审核页面依赖 changes API 进行审核操作
